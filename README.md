# WebStartUp

## Идея проекта

![](/home/billiejean/IdeaProjects/WebStartUp/images/Pasted image 20260310114836.png)

**Dispatcher** - этот микросервис не содержит бизнес логики. 
Он принимает сообщение, делает минимальную валидацию и передает его в брокер сообщений в виде json. 
Также он забирает из брокера ответы и отправляет их обратно в телеграм.

**Брокер сообщений (RabbitMQ)** - Посредник между несколькими частями системы. 
Он необходим для обеспечения асинхронности обработки и распределения сообщений между клиентами.

**Node** - Тут находится большая часть бизнес логики. 
Тут файлы будут преобразовываться в бинарный вид и грузится в базу. 
Отсюда будут формироваться запросы к другим микросервисам.

**PostgreSQL** - База данных для хранения данных о пользователях и контента в бинарном виде.

**RESTful API** - Он будет обращаться в базу данных, выгружать нужный контент 
и в POST запросе отдавать клиенту. + Обработка запросов на подтверждение регистрации.

***
## Хотелки заказчика
**База клиентов** - Предполагается, что если пользователь перейдет в наш бот из группы партнера,
где рекламируется наш бот, то необходимо это зафиксировать для формирования клиентской базы и 
дальнейшей рекламной рассылки.
